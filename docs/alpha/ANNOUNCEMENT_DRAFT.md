## Variant — long devlog intro

> **Why I built Aurion's governed mission spine first**
>
> Most "AI agent" demos show an agent doing things. The harder, more honest question is: *should it
> have been allowed to, and can you check what it actually did afterwards?*
>
> Aurion is a local-first governed AI mission system. It turns a user goal into a governed mission:
> a plan, a permission check before action, a safe action or an honest proposal, an audit trail, and a
> human-readable Mission Receipt you can open in the Command Center.
>
> This is a controlled **Demo Public Alpha** of exactly one spine —
> Goal → Mission Plan → Permission Check → AuditLedger → Mission Receipt → Command Center. I'm
> deliberately not claiming more.
> It is **not production-ready**, **not full public alpha**, and **not enterprise-ready**; live autonomy is **not complete**.
> Missing data is shown as *unavailable*, never faked green.
>
> The governance-first bet is simple: **agents should not just act; they should leave receipts.** This
> demo is the smallest honest proof of that direction. Next milestone toward full public alpha:
> greening the broad frontend test suite.
