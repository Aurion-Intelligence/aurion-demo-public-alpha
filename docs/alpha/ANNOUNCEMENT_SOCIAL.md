## Technical (3–5 short paragraphs)

> Aurion is a local-first, permission-governed AI mission system. It's mission-based: a user goal becomes
> a Mission Plan with microsteps, runs through a permission check before any action, and ends in a
> durable Mission Receipt.
>
> This Demo Public Alpha demonstrates one bounded spine end to end — goal → mission plan → microsteps →
> permission check → governed execution / dry run → AuditLedger → BlackBox trace → Mission Receipt →
> Command Center. The bounded public spine demo passes its focused tests.
>
> Governance is enforced before action: PermissionGraph allows local reads and denies external network
> and writes for the demo mission. Cost governance is advisory and read-only — no live spending. Missing
> evidence is shown honestly as "unavailable", never faked green.
>
> Honesty about scope: this is a Demo Public Alpha, not a full public alpha and not production-ready. The
> broad frontend test suite still has failures and blocks full public-alpha readiness. Live autonomy is
> not presented as complete, and governance is strong on the demonstrated spine but not yet proven
> universal across every side-effect path.
>
> If you care about auditable, local-first agents, the screenshots, demo script, and a runnable
> deterministic demo are in the repo. Feedback from technical testers is welcome — especially anything
> that looks like it's claiming more than it proves.
